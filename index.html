Project Idea

Project Summary

My Project idea is a game where you play through the different eras of gaming, starting in 1958 and moving to the 1960s, 1970s, 1980s and 1990s. You will play one round/level/game and then move on to the next. After completing the 1990s game, you will fully start the game's plot. An antagonist is destroying the earlier rounds because they want to influence the history of games and make their favourite games the only games left. It is your job to defeat the antagonist and save all gaming history. Without the old game ever existing, the newer and future matches will not have any inspiration and game mechanics made by the old games, so they disappear.

Motivation

My motivation for designing this game is, I have never seen or heard of a game like this before. I believe people need to understand and respect retro and classic games, which helped shape the gaming industry and community into what it is today. I've always had a passion and love for retro games since I was young. Even now, some of my favourite games are old games (Kirby's adventure, Mario the lost levels, Tetris, ETC). Retro game developers did not have much to work with because it was still very new; there was not much regarding coding, game mechanics and visual design. However, these developers made many game mechanics still in use today. For example, the first-ever 'Boss' in videogames was developed in 1975 in a role-playing game called dnd on the PLATO system.

Description

The game is about travelling through the years of gaming, starting in 1958, playing pong (the first game made). After beating the game (scoring 5 points), you move onto the 60s, playing Spacewar! (Win by defeating opponent) On the PDP-1 (Programmed Data Processor 1), 70s Galaxian (win by destroying all the invading aliens), 80s Tetris (get to a score of 10,000) and 90s sonic the hedgehog (complete first level). I will change the games just enough that copyright does not take the game down (changing the characters and enemies so you can still see the retro-inspired nuances). Throughout the game, little glitches will happen (visual glitches, sprites not rendering correctly). Then once you beat the Sonic the Hedgehog level, a portal will open.

The graphic will start to turn into code (matrix-like code), and sonic will turn into the player's basic form (the cursor on Windows/Mac/Linux) and walk through the portal, starting the main plot of the game, the antagonist (TBD) is destroying the old games one by one. The game developers are banding together to take the antagonist down and restore the fun with the help of the master code, a line of the first-ever written code (cout << "Hello World"), (it is that line of code because it's the first thing you write in C++). Players go and confront the antagonist with all the developers. There is a cut scene of talking with the antagonist about what they are doing is wrong and that we are here to stop them. The antagonist seems to have a change of heart and comes to have a peaceful talk. Still, it is a ploy to get the master code and wipe all the developers and protagonists out of existence. Still, the protagonist does not die, and the antagonist is stunned and retreats, fearing what the protagonist is. The protagonist wakes up in a creator where the game developers got wiped out of existence and has a moment of confusion and sadness before getting angry and wanting vengeance for the fallen comrades. The protagonist returns to the base the game developers had and gets ready for a fight with the antagonist, knowing that the antagonist has the master code and can use it to wipe the game out one by one. The protagonist sees a map leading to 4 artifacts (python-poison breath attack, HTML5-magic book with the different tags as magic spells, JavaScript- a roll dodge, java- an upgrade to the JavaScript roll dodge that make it a phase dodge that can go through enemies), weaker than the master code.

Still, each has different abilities, so the protagonist goes and searches for the artifacts and collects them. When coming back can try the new skills on the enemies (antagonists army they created). Once the artifacts are collected, the protagonist goes to the castle the antagonist has built to defeat the antagonist. Going through the castle, the antagonist has four generals (TBD). The protagonist defeats the four generals to make it to the castle's throne room, where the antagonist is waiting and destroying games. When the protagonist walks into the throne room, the antagonist is destroying a green dinosaur with a shell on its back (Yoshi from Yoshi's island) and says for the protagonist to wait while I take care of this thing (referring to Yoshi). The protagonist gets angry and rushes the antagonist. The antagonist hits the protagonist away and says," OK, you want to fight so badly? Then let us fight!" The boss battle happens. After the boss battle and the antagonist loses, the protagonist picks up the master code to try and bring back the game developers but cannot because the protagonist is a player and cannot edit the code without a terminal. So, the player (protagonist) gets sad and angry and throws the master code, hitting a trapdoor under some rubble. The protagonist goes and searches the rumble for the trap door and opens it. Going inside the trapdoor, the protagonist sees a basement with an old Macintosh SE/30 with a singular white blinking line in the top left corner of the screen. The player also sees a coffee machine and a fold-out bed. The player makes a cup of coffee, sits at the table with the Macintosh SE/30, and says," let us get started fixing things", and you can hear typing happening as the player begins to remake all the deleted games, starting with pong. Blue code creates a pong paddle in the room and flies out the trapdoor. The credits roll.

Tools and Technologies

The software needed to make the game is called flutter. It is used to create apps and games for most devices, including but not limited to android phones, iPhone, Xiaomi phones, Linux, macOS and Windows. Hardware would be most devices since it is pixel 2d and is not 3d, which means it does not need as many graphics as those 3d games. It also has the potential to run on older consoles like the GBA (Game Boy Advanced), PlayStation 1, PlayStation 2, ETC. Anything that can run 32-bits can run the game. I will also make any sprites, sprite sheets, images and backgrounds needed using an online pixel art maker like piskelapp.

Skills Required

Knowing one of three programming languages, C, C++, and Dart, requires skills. Also, an understanding of game programming fundamentals and visual design fundamentals, and knowledge of how to use flutter. It would not be difficult to obtain these skills. However, it will take time to learn the programming languages. Can do online courses to learn C, C++, and Dart. The visual design part of the project would be a bit difficult with the length of creating the Sprites, Sprite Sheets, Images and Backgrounds. And the programming part of the project will take the most time. While making multiple game mechanics, I may be able to reuse some of the lines of code from the individual game parts. For example, jumping from Sonic the Hedgehog into the main game, the collision of galaxian can be used to kill enemies in the central part of the game.

Outcome

My greatest hope for this game is that it's a hit and gets a lot of fans. Then, continuing from the momentum of this game's success, I'll develop a second game. The protagonist from the first game will switch roles and be the antagonist for the second game. The character will become mad with power, changing the games to suit their needs. Changing the timeline, the protagonist collects the same four artifacts from the first game and some new ones to try and defeat them; however, they fail to do so. Then, boom! Something happens to the antagonist. I won't reveal the entire plot before the game development, so stay tuned.
